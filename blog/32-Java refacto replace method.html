<Describe how I answer the question (references, proofs, etc.).> Starting from a method X being called and overriden:
  1. new method Y extracted from X (X → Y) 
    
		interface SizeComputer<T> {
			public long sizeOf(T data);
		}
  2. deprecate X in favor of Y
  3. replace X overrides by Y overrides
  4. replace X calls by Y calls
  5. remove X
Starting from a method X being called and overriden:
  1. new method Y calling X (Y → X) 
    
		interface SizeComputer<T> {
			public long sizeOf(T data);
		}
  2. deprecate X in favor of Y
  3. replace X calls by Y calls
  4. replace X overrides by Y overrides
  5. make Y independent of X and remove X

The key point is to never produce a Y not linked to X, otherwise we produce dead code (Y not called) or lose the overrides (Y called). TODO What when we don't see obvious X to Y or Y to X relationship?
