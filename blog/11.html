Java Without Abstract Classes

- Abstract classes are usually used to design templates, but abstractness can be annoying, like the impossibility to instantiate it through reflection.

Context

Application

Advantages

- Easier to test than abstract classes because existing operators can be used, which makes the code simpler compared to ad hoc implementation and allows to reuse already tested code. - Respect interface segregation principle. - Respect single responsibility principle. - Respect Liskov substitution principle.

Limitations

- Possibility to execute before setting any operator (i.e. missing operators found at runtime) unless constructors are used to require minimal sets of operators. As a side note, relying on setters is equivalent to using a builder, while passing through minimal constructors is equivalent to use factory methods. - Operators are additional instances, which might have a significant performance impact when a lot of instances of the operator-based class are generated.
