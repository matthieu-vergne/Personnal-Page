Different life time

Ask yourself: are all these parameters known at the same time?

If it is often the case that some parameters are known before the others, then they have a different scope. For instance, a method order(List<Item> items, PaymentMethod method) may have parameters with different scopes. Usually, you let the customer identify several items to buy, and only at the end you ask for the payment method, when the customer is ready to pay. Considering both of them at the end imposes additional effort to maintain the items until then. It would be more interesting to add the products progressively and to only ask the payment method at the end: 


	Order order = new Order();
	...
	order.add(item);
	...
	order.pay(PaymentMethod method);

Different life span

Ask yourself: do all these parameters change a the same time?

If it is often the case that some parameters are prone to change more often than others, then they have a different scope. For instance, a method log(String methodName, String message) have parameters with different scopes. The name of the method is constant for all the logs in the same method, while the message changes every time in this method. Consequently, it would be interesting to specify the method name once, at the beginning, and only the different messages at each call: 


	Logger logger = new Logger(methodName);
	
	logger.log("Start");
	...
	logger.log("Intermediate state");
	...
	logger.log("Done");
