For example, you face this situation when all your parameters do not have the same scope.

It is not about the simple need of adding a new parameter. If the parameter is meaningful and worth to add to all the calls of the method, then just add it. The point is that you want some specific customization here, without having to do it elsewhere.

Solution

<TODO>

Adapt a static method MyClass.doStatic() into a non static new MyClass().doRegular() method in order to add parameters without changing the signature of the method:

  1. Create non static method doRegular() in the class
  2. Create a static instance of MyClass (static final MyClass instance = new MyClass())
  3. Fill doRegular():
    1. Copy the content of doStatic() in doRegular()
    2. Replace the content of doStatic() by a call to instance.doRegular()
  4. Add your new parameters:
    1. Extract the element to customize in doRegular() into a final field of MyClass
    2. Add a parameter to the constructor of MyClass to fill this field
    3. Set the current value on the static instance of MyClass
  5. Keep doStatic() as a default behaviour, or replace it iteratively by doRegular():
    1. Deprecate doStatic()
    2. Iteratively replace its calls by the corresponding new MyClass(...).doRegular()
    3. Remove the deprecated doStatic() when you feel confident not to break anything

Now you can add more parameters to the constructor of MyClass without changing the signature of the methods. Only the situations needing to change these parameters need their own instance of MyClass.
